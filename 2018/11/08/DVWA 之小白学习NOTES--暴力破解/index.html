DVWA 靶场

0x00 暴力破解是啥？

一看就是和名字一样，就是用字典通过穷举法一个个去猜用户名和密码等等。说到暴力破解，自然会想到burpsuite，一般借助burpsuite来进行爆破，破解密码。

0x01 burpsuite 的爆破解说

POINT 1. burpsuite 的爆破方式

  1. Sniper

    针对得上是单一一个变量进行爆破，是最常用的爆破方式。相当于是设定一个字典，字典里面每个位置上把字典payload一个个去尝试，判断是否是符合的。总数是position*payload.(见下图)

  2. Battering ram

    这种模式是就是把字典里一个payload同时放到多个position。请求总数就是payload总数。（适合单一变量爆破）

  3. Pitchfork

    这种模式是属于至少有两个变量要爆破的类型。这种方式是一一对应的把payload-a和payload-b放到相对应的position里面进行尝试，如下图：

    uname1 –> passwd1
    uname2 –> passwd2

    尝试次数是最小的payload组（最小字典里面）的payload总数。

  4. Cluster bomb

    属于至少两个变量要爆破的类型。不同于第三种的，一一对应爆破，这个方式是混用循环方式的。比如有payload-a1、payload-a2、payload-b1和payload-b2，进行这种模式见下图：

    uname1 –> passwd1
    ​ –> passwd2
    uname2 –> passwd2
    ​ –> passwd1

POINT 2. burpsuite 自带字典

burpsuite自带字典是大多是常用的弱密码集和sql语句集。

p.s. burpSuite里面要使用自带的字典进行爆破的话需要使用pro版本。

POINT 4. 彩虹表

这个不是burp自带的字典，是一个用于加密散列函数逆运算的预先计算好的表, 为破解密码的散列值（或称哈希值、微缩图、摘要、指纹、哈希密文）而准备。一般主流的彩虹表都在100G以上。

0x02 LOW 级别

POINT 1. 源码分析

  • PROBLEM1: isset() 判断php的变量是否是为空，是否设置了，返回布尔值。这里只是设置了isset检测判断变量是否设置了，没有做防爆破机制，可能导致暴力破解。
  • PROBLEM2: username没有做过滤，可能导致SQL注入 。
  • PROBLEM3: passwd直接接受参数值，未过滤，之后再md5加密，防止对pass部分进行SQL注入，并不能防止username上面的sql注入，毕竟没有对username上面做过滤。

0x03 Medium 级别

POINT 1. 源码分析

  • PROBLEM1:isset()判断是否提交，没有做防爆机制
  • PROBLEM2和PROBLEM3:username和password没有做防爆机制，会导致暴力破解；同时输入上的过滤是通过mysqli_real_escape将特殊字符（x00，n，r，，’，”，x1a）进行转义了，基本防止了SQL注入。pass部分在后面又用md5进行加密了，所以基本上通过对输入进行编码加密防御了sql注入。

POINT 2. payloads

第一种：暴力破解

由于源码中设立了mysqli_real_escape_string()函数的原因，最基本的sql注入方式会失效（但是mysqli_real_escape_string()可以绕过，绕过后如何注入见后面第二种方法）。可以用burp直接暴力破解。

同low等级中的配置好burp基本就可以开始爆破了。刚才源码分析的一点说到了sleep(2)延迟了错误情况的返回页面，那么在爆破的时候爆破的时间肯定会因此而受到影响，时间明显变长。

0x05 Impossible 级别

POINT 1. 源码分析

  1. 添加CSRF防御机制Anti-CSRFtoken
  2. 添加stripslashes()去除反斜杠
  3. 输入使用mysqli_real_escape_string()转义控制字符
  4. 密码md5加密
  5. 记录登录错误次数。
  6. 采用PDO机制防御sql注入

POINT 2. payloads

0x06 Summary: low, medium, high 防御手段

POINT 1. 各个等级用了哪些防御手段？

各个等级的防御手段是各不同的：

  1. 添加Anti-CSRFtoken预防无脑爆破的校验手段
  2. 添加sleep延时，降低爆破速度
  3. 增加登录错误的机制
  4. 采用PDO机制防御sql注入，因为不能用他的扩展执行任何数据库的数据，而sql注入便是“闭合”原来的语句，根据逻辑构造自己的语句放进去并执行。
  5. mysqli_real_escape_string()转义控制字符
  6. stripslashes()去除反斜杠

后面出现的新的防御手段，我更多系统学习了后，详情后期补充。
