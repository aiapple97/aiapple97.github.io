    0x00 文章的初衷

    写这篇文章不是因为我是搞硬件要画板子需要装这个软件，而是因为一个朋友需要这个软件，但是一直装不好，让我帮下忙搞定下。刚开始觉得纳闷，搜索了下Candence的安装教程，发现无论是哪个版本，安装起来对于新手来说确实是很难搞定的，而且网上流传的安装版本基本上是默认路径的，并且安装过程说的不明不白。所以这里给大家写个简单易懂，详细的安装破解教程。

    0x01 下载Cadence SPB 16.6 和 破解文件

    网上的资源很多，但是这里为了方便还是给出文件的资源供大家下载：

    百度云:https://pan.baidu.com/s/1eS8LbVG

    see password in footer

    链接里面有：Cadence SPB 16.6 安装包、破解文件压缩包和配件包，确定下有没有漏掉什么哈。

    0x02 安装Cadence

    STEP 1. License Manager 证书系统安装

    把刚才下载的各个压缩包给解压后，然后打开Cadence的安装包，找到Setup.exe文件：

    双击点开后，点击第一个License Manager：

    一直选择next到安装路径的时候不一定要选择默认路径，也不一定要安装像安装路径里面一样建立名字为LicenseManger的安装文件夹，但为了讲解方便建立这个文件夹。（我这里的证书系统安装路径是E:\CadenceSPB16.6\LicenseManager）

    这个安装完后出现了要求添加License文件的时候点击cancel

    弹出对话框，表示确认你是否是同意退出，点击“是”，然后跳转至结束安装页面，点击“finish”。之后选择第二个选项Product Installation

    STEP 2. Product Installation 产品安装

    同安装上面的证书安装一样，首先点击next进行默认安装一直到选择安装路径。这个路径对应的是下图的第一个红框，而第二个红框的路径是安装好软件使用时候保存工程文件的工作目录的路径。这个可以随意设定，根据自己的喜好。

    注意：这里的安装路径和刚才安装证书系统的路径最好在一个大的文件夹下，也就是根目录下，按照刚才的路径来说（证书系统的路径是E:\CadenceSPB16.6\LicenseManager），所以这里的产品路径下应该是E:\CadenceSPB16.6\SPB_16.6(建议产品文件夹还是叫 SPB_16.6 )。


    上图中的第三栏是填写端口号和计算机全名的。端口号默认为5280，如何查看计算机全名呢？
    打开：控制面板–系统和安全–系统选项下面查看该计算机的名称（点击）

    点击右侧的高级系统设置：

    弹出对话框后复制计算机全名：

    直接填写到刚才的第三栏里面就可以了。然后开始安装，安装过程很慢，需要耐心等很久哈。一直到安装完毕后，点击finish即可安装完毕，finish页面的两个选项都是关于产品特性介绍的，不想了解的话就不用选择。

    0x03 破解Cadence

    STEP 1. 移除安装系统文件

    找到”E:\CadenceSPB_16.6\SPB_16.6\tools\pspice（相对于你：\SPB_16.5\tools\pspice）”目录下的orSimSetup.dll,然后剪切移动到其他的文件夹或地方下，不然导致不能成功破解。

    STEP 2. 破解1

    解压破解包Cadence_SPB16.60_Crack.zip后打开文件夹,选择在/aspirin/Cadence/LicenseManager文件夹，复制pubkey.exe,pubkey,LicenseManager.bat文件到路径\SPB16.6\LicenseManager下（实际中复制到你的路径下证书系统的根目录下）

    然后选中上图第一个bat文件，右键选中编辑选项，在最后一行增添一行pause。（目的在于不让此脚本运行完后或者运行错误就自动闪退，可以查看清楚哪里出错了）运行LicenseManager.bat文件，直到如下图为止：

    STEP 3. 破解2

    返回破解文件夹，在\aspirin\Cadence\SPB_16.6\tools中将pubkey.exe,pubkey,Tools.bat文件复制到\SPB16.6\tools下面

    复制完毕后，和上一步破解一样，先编辑Tools.bat文件添加一行pause。然后运行Tools.bat文件，直到出现停止。

    注：
    1. 如果上述的Tools.bat文件运行过程中出现can not open或者是 diff pubkey等字样时不必惊慌，文件还是能够破解的，继续运行下面的步骤。
    2. 如果出现Total 0 files were changed，查看下面的FAQ的Q2

    STEP 4. 生成license.lic文件

    回到破解文件下，在aspirin\Cadence\LicGen里面编辑LicGen.bat文件，添加pause一行。等生成完毕后，会生成一个license.lic文件。

    用记事本打开license.lic文件，修改文件第一行：

    修改完毕后，把这个文件放到路径为CadenceSPB16.6\LicenseManager下面。

    0x04 检查环境配置

    虽然上面的安装和破解过程已经完毕，但是需要检查环境配置是否完毕。打开控制面板->系统与安全->系统->高级系统设置（详细步骤见前面查看计算机全名的过程）。在高级选项卡点击环境变量, 新弹出窗口中在用户变量中查看是否有以下选项：

    有的话就表示STEP 5和STEP 6的破解成功。

    0x05 重启服务器

    打开菜单，找到Cadence里面的LMTools，双击运行。选择ConfigSerices选项卡，在path to license file点击Browse，将License.bat换成License.lic，然后点击save service

    又选择Start/Stop/Reread选项卡，先单击Stop Servi ce停止服务，然后单击Start Service。停止服务状态显示stopping server,开启服务状态start server successfully表示已重启成功。

    0x06 FAQ

    Q1:
    检查环境变量的时候没有看到CDS_LIC_FILE变量值是自己的计算机名字，而是看到他的路径？
    A:
    这个问题是可能是STEP 5或STEP 6没有在server或client方向里激活起证书，但是证书存在，可以自己手动建立一个：点击查看环境变量页面的用户变量下面的新建选项，填上：

    Q2:
    STEP 6 完毕后显示无法加载：Unable to restart Cadence License Server with the new license file ‘F:\Cadence\LicenseManager\license.dat’.
    A:
    1.可能是没有添加成环境变量，参照Q1手动添加。
    2.STEP 5没有成功，重新开始做STEP 5。
    3.前面运行Tools.bat失败，出现了第一大段后面有Total 0 files were changed. 把cdslmd.rar解压包解开后，将cdslmd.exe文件粘贴到\CadenceSPB_16.6\LicenseManager下，然后重新运行一遍，继续操作下去。
    Q3：
    STEP 5完毕后显示：ERROR: Unable to update the CDS_LIC_FILE license path environment variable.
    A:
    同Q2的第三个方法。
