    0x03 Medium 级别

    POINT 1. 源码分析

    1. isset()判断是否提交，提交是否置空

    2. user和pass使用了mysqli_real_escape_string()，对指定的字符会进行转义。

      PS:

      >

      $link 是由mysqli_connect() 或 mysqli_init()返回值。

      这个是用来指过滤了空字符，单引号，双引号，换行，tab，x1a,等等其他的控制字符，将他们转义了变成普通的字符，失去本身的特殊意义。并不转义%和_不过在编码上有要求：

      这个函数默认的数据库字符集是utf8。

      ​

      不同于mysql_real_escape_string()函数的漏洞，详情见：

      PHP <=5.3 - mysqli_real_escape_string()信息泄漏漏洞

      ​

      在求mysql_real_escape_string()防注入详解这个问题中，看到了默认数据库字符集建议还是使用utf8或者是utf8mb4，但是如果是GBK并且版本是MySQL5.5.37以下版本,就可以绕过这个函数的单引号转义（还是有很大一批在使用MySQL5.0）.但是单引号和单引号里面的字符串过滤的情况并不是所有由这个引起的，比如只是过滤了单引号字符串里面的东西，从而整个语句失效，比如空格，有的时候使用hex, 0x开头的hex编码，0x后面的内容被视为是字符串，区别于语句。

      具体详情可以查看：mysql_real_escape_string绕过和笔记：mysql_real_escape_string绕过

      PS: 不过确实有点晕这两个函数的区别，有点混乱的。

    3. pass密码由md5加密，防止了通过password参数进行sql注入，可以考虑爆破

      freebuf的文章说sleep(2)不算是防爆破机制关于爆破

    4. 语句格式带单引号的。

    5. 语句最后的sleep(2)延迟语句，会导致爆破的时候失败了时间延迟>=2s

    POINT 2. payloads

    由于源码中设立了mysqli_real_escape_string()函数的原因，一般的sql注入方式失效。但是由于md5的防爆机制基本没有（虽然有个sleep(2)），考虑burp爆破.

    同low等级中的配置好burp基本就可以开始爆破了。刚才源码分析的一点说到了sleep(2)延迟了错误情况的返回页面，那么在爆破的时候爆破的时间肯定会因此而受到影响，时间明显变长。当然，不管这个也可以继续爆破出来，但是如果不想弄这么久该怎么做呢？(下面的这个方法是DVWA - Brute Force (low, medium, high) by jchalex)

    0x05 Impossible 级别

    POINT 1. 源码分析

    1. 添加CSRF防御机制Anti-CSRFtoken
    2. 添加stripslashes()去除反斜杠
    3. 输入使用mysqli_real_escape_string()转义控制字符
    4. 密码md5加密
    5. 记录登录错误次数。
    6. 采用PDO机制防御sql注入

    POINT 2. payloads

    这个级别感觉一般是破解不了，如果是大型网站的话可以从其他方面入手攻入了。

    0x06 Summary: low, medium, high 防御手段

    POINT 1. 各个等级用了哪些防御手段？

    各个等级的防御手段是各不同的：

    1. 添加Anti-CSRFtoken预防无脑爆破的校验手段
    2. 添加sleep延时，降低爆破速度
    3. 增加登录错误的机制
    4. 采用PDO机制防御sql注入，因为不能用他的扩展执行任何数据库的数据，而sql注入便是“闭合”原来的语句，根据逻辑构造自己的语句放进去并执行。
    5. mysqli_real_escape_string()转义控制字符
    6. stripslashes()去除反斜杠

    后面出现的新的防御手段，我更多系统学习了后，详情后期补充。
